## Content Development Process

### Creating Articles

1. **Create GitHub Issue** - One per article with roadmap details
2. **Draft Content** - Create `.md` file in `src/content/blog/`
3. **Use Frontmatter Template**:
   ```yaml
   ---
   title: "Article Title"
   description: "Brief description (50-60 chars)"
   pubDate: 'Dec 17 2025'
   heroImage: "/images/article-hero.jpg" # optional
   ---
   ```

4. **Write Content** - Markdown format, clear sections, personal narrative + practical insights
5. **Add Tags** - Use existing category tags from issues
6. **Review & Publish** - Test locally, commit to branch, create PR

### Blog Post Format

- **Opening**: Hook with personal context or relevance
- **Main Sections**: 3-5 substantial sections with headers
- **Examples/Stories**: Real-world scenarios, personal experiences
- **Practical Takeaways**: Actionable insights or lessons
- **Closing**: Connection to broader interests or future posts
